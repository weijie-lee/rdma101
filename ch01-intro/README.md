## 1.1 传统TCP/IP通信的问题

### 计算机网络通信基础

在了解RDMA之前，我们先回顾传统网络通信的工作方式：

```
┌─────────────────────────────────────────────────────────────────┐
│                        应用层                                    │
│                     (你的程序)                                   │
└─────────────────────────────────────────────────────────────────┘
                              │ write()/send()
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                        传输层 (TCP/UDP)                         │
│                   内核协议栈处理                                 │
└─────────────────────────────────────────────────────────────────┘
                              │ 
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                        网卡驱动                                 │
│                   (复制到DMA缓冲区)                             │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                          网卡                                    │
│                   (发送到网络)                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 传统通信的数据拷贝

```
用户空间 ──copy──▶ 内核缓冲区 ──copy──▶ 网卡DMA缓冲区 ──发送──▶ 网络
     ◀──copy── ◀──copy── ◀────────────────────────────────────────
                              接收
```

**数据经历了多少次拷贝？**
- **发送**：用户空间 → 内核空间 (1次)
- **接收**：内核空间 → 用户空间 (1次)
- **总计**：至少 **2次** 拷贝

### 传统通信延迟来源

| 操作 | 典型延迟 | 说明 |
|------|----------|------|
| 系统调用 | 1-2 μs | 从用户态切换到内核态 |
| 数据拷贝 | 1-2 μs | 内存复制开销 |
| TCP处理 | 5-10 μs | 协议栈处理 |
| 中断处理 | 1-5 μs | 硬件中断 |
| 网络传输 | 100s-1000s μs | 取决于物理距离和带宽 |

### 传统通信的问题总结

1. **CPU参与** - 每次通信都需要CPU介入处理
2. **数据拷贝** - 数据在不同内存区域间多次拷贝
3. **内核瓶颈** - 所有数据必须经过操作系统内核
4. **延迟高** - 多次上下文切换增加开销

## 1.2 RDMA 核心优势

### 什么是RDMA？

**Remote Direct Memory Access (RDMA)** - 远程直接内存访问

允许一台计算机直接访问另一台计算机的内存，无需对方CPU和操作系统的参与。

### 四大核心优势

| 优势 | 说明 |
|------|------|
| **零拷贝** | 数据直接从一个内存区域传输到另一个，无需CPU参与 |
| **内核旁路** | 绕过操作系统内核，直接与网卡通信 |
| **低延迟** | 端到端延迟可低至1-2 μs |
| **低CPU占用** | 通信不消耗CPU资源 |

### RDMA vs TCP/IP 延迟对比

```
TCP/IP:  10-100 μs (包含多次拷贝和内核处理)
RDMA:   1-5 μs   (直接内存访问)

性能提升: 10-20倍
```

### RDMA 数据传输

```
┌─────────────────────────────────────────────────────────────────┐
│                    传统 TCP/IP 通信                             │
│                                                                 │
│  发送方                           接收方                         │
│  ┌─────┐                        ┌─────┐                        │
│  │CPU  │ copy                   │CPU  │ copy                  │
│  └──┬──┘   │                 ◀──└┬──┘   │                     │
│     │      ▼                        │      ▼                    │
│  ┌──▼───────────────────────────────▼──┐                        │
│  │         内核协议栈                    │                        │
│  └──────────────────────────────────────┘                        │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    RDMA 通信                                     │
│                                                                 │
│  发送方                           接收方                         │
│  ┌─────┐                        ┌─────┐                        │
│  │CPU  │ ──RDMA Write────────▶│内存 │                        │
│  └─────┘   (无CPU参与)          └─────┘                        │
└─────────────────────────────────────────────────────────────────┘
```

## 1.3 三种RDMA协议

### 1. InfiniBand (IB)

- **特点**：原生RDMA协议，专为高速网络设计
- **带宽**：HDR 400Gbps, NDR 800Gbps
- **需要**：专用IB交换机
- **优点**：性能最高
- **缺点**：需要专用硬件

### 2. RoCE (RDMA over Converged Ethernet)

在以太网上运行RDMA，是目前最流行的方案。

**版本**：
| 版本 | 说明 |
|------|------|
| RoCE v1 | 需要无损网络 (Data Center Bridging) |
| RoCE v2 | 支持路由，基于UDP (端口号4791) |

**优势**：
- 兼容现有以太网设备
- 可使用标准以太网交换机
- 支持跨路由

### 3. iWARP (Internet Wide Area RDMA Protocol)

- **特点**：基于TCP/IP的RDMA
- **优势**：可穿越防火墙、NAT
- **性能**：比IB和RoCE稍差
- **使用场景**：广域网

### 协议对比

| 特性 | InfiniBand | RoCE v2 | iWARP |
|------|------------|---------|-------|
| 性能 | 最高 | 高 | 中 |
| 兼容性 | 专用硬件 | 以太网 | TCP/IP |
| 路由支持 | 是 | 是 | 是 |
| 穿越防火墙 | 否 | 否 | 是 |
| 典型带宽 | 400/800 Gbps | 100/200 Gbps | 40/100 Gbps |

### 选择建议

| 场景 | 推荐协议 |
|------|----------|
| 高性能计算 (HPC) | InfiniBand |
| 数据中心 | RoCE v2 |
| 需要跨广域网 | iWARP |

## 1.4 RDMA 核心术语

### Queue Pair (QP) - 队列对

RDMA通信的基本单元，由两部分组成：
- **SQ (Send Queue)** - 发送队列
- **RQ (Receive Queue)** - 接收队列

```
┌─────────────────────────────────────┐
│            Queue Pair                │
│  ┌───────────┐   ┌───────────┐      │
│  │    SQ     │   │    RQ     │      │
│  │(发送队列) │   │(接收队列) │      │
│  └───────────┘   └───────────┘      │
└─────────────────────────────────────┘
```

### Memory Region (MR) - 内存区域

- 注册到RDMA设备的虚拟内存区域
- 包含：**虚拟地址**、**物理地址**、**访问权限**

```
用户虚拟地址 ──注册──▶ RDMA设备 ──转换──▶ 物理地址
```

### Protection Domain (PD) - 保护域

- 用于隔离不同应用的资源
- QP和MR必须属于同一个PD

### Completion Queue (CQ) - 完成队列

- 存储通信操作完成通知
- 包含操作状态、错误信息

### Work Request (WR) - 工作请求

- 提交到队列的操作请求
- 描述要执行的操作及参数

### Work Completion (WC) - 工作完成

- 操作完成后的状态返回
- 包含：状态、操作类型、ImmData等

## 1.5 RDMA 操作类型

| 操作 | 说明 | 需要对方配合 | 典型用途 |
|------|------|--------------|----------|
| **Send/Recv** | 传统消息传递 | 是 | 消息通知 |
| **RDMA Write** | 直接写远程内存 | 否 | 数据推送 |
| **RDMA Read** | 直接读远程内存 | 否 | 数据拉取 |
| **Atomic** | 原子操作(CAS/FAA) | 否 | 计数器、锁 |

### 操作时序图

```
Send/Recv:
  A                          B
  │───post_recv()───────────▶│
  │                          │
  │───post_send()───────────▶│
  │◀──────completion─────────│
  │                          │
  │◀───post_recv()──────────│

RDMA Write:
  A                          B
  │───RDMA Write───────────▶│ (直接写B的内存)
  │◀──────completion─────────│

RDMA Read:
  A                          B
  │───RDMA Read─────────────▶│
  │◀─completion (数据在A)────│ (B无感知)
```

## 1.6 编程框架

### libibverbs (Verbs API)

- RDMA编程的底层API
- 硬件厂商通用接口
- Linux原生支持

```c
#include <infiniband/verbs.h>
```

### RDMA CM

- 基于连接的通信管理API
- 建立在libibverbs之上
- 提供更易用的接口

```c
#include <rdma/rdma_cma.h>
```

### 对比

| 特性 | Verbs API | RDMA CM |
|------|------------|---------|
| 复杂度 | 高 | 低 |
| 控制力 | 完整 | 有限 |
| 适用场景 | 高性能 | 简化开发 |
